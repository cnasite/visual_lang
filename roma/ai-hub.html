Asite Intelligence · Roma register

One instrument, now intelligent.

Asite Intelligence isn't a separate product bolted on — it's the same achromatic surface, the same restraint, doing more. It reads the register, drafts the response, flags the risk, and always shows its working. No AI purple, no robots, no gradients: the spark burns red only when it is live.

01 — The layer

The shared vocabulary.

One set of words across design and engineering, so we describe the same things the same way.

Intelligence

The brand name for the layer — Asite Intelligence, never "the AI", "the bot", or "Copilot". A competent colleague that reads the project and acts on it.

Run

One unit of work an agent performs, start to finish. Has a run id, a state, a model, and telemetry — tokens, latency.

Tool call

A function the agent invokes mid-run — a search, a lookup, a write. Rendered as a quiet mono readout, not a raw log.

Reasoning

The agent's working. Always opt-in and quiet — collapsed, dimmed, secondary to the answer.

Citation

A traceable source for any claim about project data — an RFI, document, or model. Required, not decorative.

Confidence

How sure the agent is, shown only when real and actionable. Low confidence always offers a path to verify.

02 — Principles

How intelligence behaves here.

Six rules keep the AI layer feeling like the same trustworthy instrument, not a different app wearing the logo.

Only the live thing burns

Asite Red is the only accent, carried by whatever is running or streaming right now. Queued, idle and finished states stay monochrome and dot-first.

The answer leads, the working follows

Lead with the conclusion in plain language. Reasoning and telemetry sit beneath it, collapsed and dimmed. Never make the thinking louder than the answer.

Provenance or it didn't happen

Every claim about project data carries a citation to source. On a construction programme, a confident answer without a source is a liability.

Calm under load

If three agents run at once the surface still reads quiet — only active dots pulse, in one hue. Motion is reserved for the single live element.

Honest about uncertainty

Show confidence only when it's real. Admit limits plainly and offer a way to verify. A fabricated number erodes trust faster than none.

No new colour

No AI purple, no gradients. Intelligence is expressed through the existing system — the spark, the orb, the pulse — not a second brand.

03 — The mark

The spark & the orb.

Intelligence has one glyph: a four-point spark — the setting-out cross, sharpened to a star. Filled red when it is the live agent; a thin gray line when it's a quiet label. The orb is the spark at the datum of concentric survey rings. Never invent a robot, a speech bubble, or a pair of eyes.

The spark — live

Filled red. Marks the agent and the act of intelligence — the one place the accent belongs in this layer.

The spark — quiet

1.4px line, gray. For labels, menu items, and any non-live reference to Intelligence.

The orb

The spark at the datum of concentric rings. For entry points, avatars, and hero moments.

03c — Motifs

The intelligence motifs.

Site gestures, extended to the AI layer. One per composition — they never stack.

The Pulse

The live signal — a burning point with expanding rings. Only for what is running or streaming.

The Constellation

Orchestration as a survey network — sources and steps as nodes, the agent as the spark at the centre.

The Stream

Tokens arriving — a dashed leader resolving to one bright point. The caret is its smallest form.

The Gauge

Confidence as an instrument dial — a quiet arc with a needle. Reads at a glance, never shouts a number.

07 — Where it lives

Two ways Intelligence shows up.

A side chat you summon to ask about what's on screen — and a Coworker that works alongside you inside a task. Both wear the same spark; both are assistive and access-scoped, never the final decision.

09 — Voice

How Intelligence speaks.

A competent colleague, not a chatbot. Leads with the answer; shows working only on request; cites sources; admits uncertainty plainly. No emoji, no marketing superlatives.

In register

"Rev A is blocked pending RFI-0847. The register shows Rev B as current — I can flag the mismatch for the document controller. Sources attached."

Out of register

"Great question! 🎉 I'd be absolutely delighted to help you supercharge your workflow! Let me work my magic on those files for you…"

10 — The short list

Do & Don't.

Do

  • Let only the running/streaming element carry red; keep queued, idle and done monochrome.
  • Lead with the answer; collapse reasoning and telemetry beneath it.
  • Cite a source for every claim about project data.
  • Show confidence only when it's real, and offer a way to verify.
  • Return field-level, reviewable changes — Coworker applies nothing until asked.

Don't

  • Introduce AI purple, gradients, robots, eyes, or speech-bubbles.
  • Make the reasoning or telemetry louder than the answer.
  • Paint every step red, or pulse more than the one live element.
  • Show a fabricated confidence number, or a claim without provenance.
  • Apply a Coworker change silently, or use a filled red error banner.
