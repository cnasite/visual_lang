BlueBand · Locked

Sharp. Structured. Locked.

Purely enterprise. Three colors — blue, black, white — and one 4px radius. Planes are organised in space and lifted on soft shadow, never boxed in outlines; headings hold apart from content by air alone; filled sections carry the hierarchy. One blue does every signal job. Emphasis is weight, not hue. The system reads like a set of contract documents: exact, repeatable, signed off.

01 — Logo

Black carries it, blue points

On black, the mark runs white with a blue arrow — the only place brand and signal meet. On white it is solid black, arrow included: the mark is structure, not decoration, and it never competes with the one blue on the page.

02b — Color, in the world

Where the blue comes from

Enterprise blue isn't a software default here — it's the color this industry already trusts with its record. Three objects carry the palette's whole personality.

RecordThe blueprint

Before the CDE there was the cyanotype: white lines on engineering blue, the one copy everyone built from. In product: deep blue is where the system states the record.

CommitmentThe chalk line

Pinned at both ends, snapped once — straight, instant, and not up for debate. In product: blue marks the committed path. The action, the active place, the selected row.

Sign-offContract ink

Wet signatures go down in blue so the original can be told from the copy. In product: pressing blue is signing — it commits, and the system holds you to it.

07 — Illustration

Drawn with the contract pen

Illustration is the icon grammar at scale: 1.5px strokes, square caps, mitered corners. Four modes on two axes — Line or Blueprint, in elevation or isometry. Line modes explain on white; Blueprint inverts to the deep blue field — the record itself. Never mix modes in one composition.

Mode 01 — Line elevation The site, surveyed

Scenes built from what a site contains — towers, cranes, floor lines. Built work is solid black; planned work is dashed gray; the blue flag sits on the one committed element. For heroes, empty states, storytelling.

Mode 02 — Line isometry The register, as object

Data given mass — slabs on a platform, faces stepped through the grays. States read as material: solid is done, blue-tinted is active, dashed is planned. For product moments and architecture diagrams.

Mode 03 — Blueprint The record itself

White lines on the deep blue field — the one place the system inverts, straight from the provenance. The bright marker is the live point. For covers, dividers, hoardings, moments of gravity.

Mode 04 — Filled isometry The object, made solid

Faces filled from the structural grays, outlines gone; the committed slab goes solid blue with darkened sides. For app icons, launch art, roadmap monuments.

One pen

1.5px strokes, square caps, mitered joins — identical to the icons. A rounded line is off-brand.

Dashed means not yet

Future floors, planned slabs, pending planes — anything unbuilt is drawn in dashes.

Blue is committed

One blue element per scene — the flag, the active slab, the live point. Never two.

Blueprint inverts, alone

White-on-deep-blue is reserved for record moments. It never mixes with the line modes.

11 — Tables & lists

Registers, not spreadsheets

The table is the product. Locked rows are 44px, single-line, with grid references and disciplines as columns; the toolbar is welded to the table; selection is a blue tint. Proven at 2,481 rows in the register before any pattern ships elsewhere.

The toolbar holds the line

Search, filters, views, columns and export on one sticky row, with the header row pinned beneath — tools never scroll away from the data they act on.

Single-line, 44px, locked

Grid ref and discipline are columns, not metadata lines. At enterprise scale, density is the comfort — and mono keeps the references scannable.

Status ranks the rows

Tinted badges — green moving, amber waiting, red stopped, gray inert — let 2,000 rows sort themselves at a glance without a single loud pixel.

Lists are registers in miniature

Action queues, RFI groups, activity feeds — same anatomy: badge, title, mono references, keyboard affordance. One grammar, learned once.

11b — Data entry

Every value has a correct field

Entry is the one place the product asks instead of reads, so it asks precisely. The type of the value chooses the control — text takes the sans, refs and quantities take the mono, decisions get only the options that exist. 13 — Principles

The lock

Five rules, non-negotiable.

Three brand colors, plus state

Blue acts, black structures, white carries. Green, amber and red exist only as status tints — state, never decoration. Any other hue on screen means the system is broken.

One radius: 4px

Corners soften just enough to feel machined, never round. The same 4px on every button, tag, plane and avatar — pills, circles and mixed radii read as errors.

Space is the container

No outlined boxes, no scaffolding rules. White planes lift on soft shadow and hold apart by measured gaps; 1px rules survive only inside tables and lists. A border drawn around a white box is a defect.

Blue means act

If it's blue, you can do something: the primary action, the link, the selected row, the active tab. Blue never decorates — a blue thing that does nothing is a defect.

Mono measures

Refs, grids, revisions, counts, figures, timestamps — IBM Plex Mono, always. Numbers are measurements, and measurements don't wear display type.

The register is the proof

Every rule above is tested at 2,481 rows before it ships anywhere else. 44px single-line rows, columns locked to the grid, the toolbar welded to the table.
