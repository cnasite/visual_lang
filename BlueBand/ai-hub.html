BlueBand · AI in the product

Two layers over one instrument.

Asite is a record. Over that record sit two distinct layers, each with one job. Asite Intelligence is the knowledge layer — it knows the record, explains it, and surfaces what matters. It never touches the work. Asite Agent is the action layer — it does a thing it was told to do, and only shows up when there is a thing to do. Same blue, same square geometry, two marks that never get confused. This page is how each one appears inside the product.

01 — Two marks, two jobs

Knowledge and action never share a mark

The split is the whole system. If a surface only knows things, it wears the square-spark and it cannot act. If a surface does a thing, it wears the orbit and it only exists because there is something to do. Get this wrong and the user can't tell whether the product is about to write to the record.

Knowledge layer

Asite Intelligence — the square-spark

A square holding an inner spark with four ticks: the record, lit. It knows, explains, answers and surfaces insight, citing where it read. It never writes, never commits, never moves a thing on the record on its own.

Action layer

Asite Agent / Coworker — the orbit

A filled blue core inside a thin diamond track, with three small square satellites at work on it: an agent doing a job. It appears only when there is an instructed action to run — draft, check, route, reconcile — and disappears when the job is done.

DraftsChecksRoutesReconcilesNever lingers
The rule that separates them

Intelligence is allowed everywhere, all the time, because it only ever reads. It can sit ambient on any screen — it changes nothing.

And holds them apart

The Agent is allowed only where there is a job, because it writes. No job, no orbit — it is never decoration and never lingers as an idle presence.

Both marks are built from the same pen as the iconography — 1.5px strokes, square caps, mitered corners, no circles or rounded forms anywhere. One blue carries both.

02 — Pattern 1 · global

The Intelligence blob

A floating assistant pinned bottom-right of every screen. It is a knowledge tool: it pulls context from across the record — registers, drawings, RFIs, workflow logs — to answer a question, whether that question is about the screen you're on or anything else in the product. It is the one place Intelligence has its own surface, because you came to it. It still only reads; nothing it shows writes to the record.

A — Closed trigger pill

At rest it is a quiet white pill on soft shadow, square-spark in blue, a mono ⌘I hint. It sits above the work, never a row in the stack.

03 — Pattern 2 · in context

The insight banner

An inline insight shown inside the real content — a dashboard, a report, a card, a workflow view. It states one thing relevant to what you're looking at, and may or may not carry a CTA. The strict rule is restraint: it has no background at all. No fill, no gradient, no pastel wash. It reads as a quiet line of text set among the data, distinguished only by a small square-spark marker and a thin blue left rule. It never competes with the numbers it comments on.

04 — Pattern 3 · action

Asite Agent / Coworker

The action layer, wearing the orbit. It only exists when there is a job to do — something the user instructed, that the Agent then carries out against the record. It shows up in three forms: a floating chat that runs the task, a CTA embedded where the work lives, and a small inline trigger. In all three the orbit mark says the same thing: this will do something.

05 — Which one, when

One ruled matrix, no cards

Three surfaces, two layers. The question is always the same: does the user need to know something, or do something — and did they come to it, or did it come to them.

06 — The discipline

The lock on the AI layer

Everything above reduces to a short set of rules. Break one and the layers blur — and the user stops trusting whether the product is about to write to the record.

Do
  • Let Intelligence explain, never act. It reads the record, answers, and cites its sources. Its CTAs open, navigate and filter — they never commit.
  • Show the Agent only when there's a job. The orbit appears with an instructed action and disappears when the job is done.
  • Keep the insight banner backgroundless. A square-spark marker and a thin blue left rule, set among the real content. Nothing more.
  • Hold the commit on a person. The Agent drafts, checks and routes; the sign-off that writes to the record is always a human press.
  • Use one blue and square motifs. Square-spark for knowledge, orbit for action — same pen, square caps, mitered joins, one signal blue.
  • Set every figure in mono. Refs, location codes, revisions, counts and durations are IBM Plex Mono, in prose and in panels alike.
Don't
  • Don't let Intelligence write. If a surface commits, moves or issues anything, it is the Agent and must wear the orbit — not the spark.
  • Don't leave the Agent lingering. No idle orbit as ambient decoration. No job, no mark.
  • Don't give the insight banner a fill. No background, no gradient, no pastel wash, no box. The moment it grows a surface it has become a blob in the wrong place.
  • Don't reach for a second hue. No orange, no aurora gradient, no decorative tints. Status tints are for traffic states only.
  • Don't round the marks. No circles, no soft orbits, no rounded sparks. The square language is the brand.
  • Don't confuse the two panels. The blob panel is light; the Agent panel runs steps. Neither is a dense blue plane.
